# Proton-Shielded-Automation (Proton VPN + Ubuntu)

Docker上のUbuntu環境からProton VPN（OpenVPN）を経由して、IPアドレスを秘匿しながらWebスクレイピングを行うためのパッケージです。

## 1. 概要
このシステムは、ホストマシンの環境を汚さずに、VPNトンネルを介したネットワーク通信を実現します。特に、Dockerコンテナ内でのDNS解決の失敗やMTU（パケットサイズ）によるフリーズといった、VPN構築特有のトラブルを解決済みの構成となっています。

## 2. ディレクトリ構成
```
.
├── docker-compose.yml  # コンテナのネットワーク・権限・DNS定義
├── Dockerfile          # Ubuntu + Python環境のビルド
├── entrypoint.sh       # VPN接続確立の制御、最適化、実行スクリプト
├── (1)                 # スクレイピング実行用Pythonスクリプト
└── vpn/                # VPN設定ディレクトリ（要作成）
    ├── (2)             # Proton VPNからダウンロードした設定ファイル
    └── pass.txt        # OpenVPN専用認証情報
```
(1)には同リポジトリの任意のスクレイピング用プログラム、(2)には下記3章ステップ2でダウンロードした.ovpn 設定ファイルを配置してください。

## 3. セットアップと実行
### ステップ 1: Proton VPN 認証情報の取得
1. Proton VPNの通常のログインパスワードではなく、OpenVPN専用の認証情報が必要です。
2. Proton VPN Dashboard にログインします。
3. 左メニューの Downloads もしくは Account セクションにある 「OpenVPN / IKEv2 username」 と 「OpenVPN / IKEv2 password」 を確認し、メモします。

### ステップ 2: 設定ファイルの配置
1. プロジェクトルートに vpn フォルダを作成します。

2. Proton VPN公式サイトから .ovpn 設定ファイルをダウンロードし、vpn/ 内に保存します（ファイル名は任意ですが、entrypoint.sh 内の記述と一致させてください）。

3. vpn/pass.txt の中に、以下のように2行で記述します（余計なスペースや文字を入れないでください）。

```
あなたのOpenVPNユーザー名
あなたのOpenVPNパスワード
```

4. entrypoint.shの22行目を、スクレイパーを起動する時のコマンドに置き換えます。
例:
```Bash
scrape_kpedia.py https://www.kpedia.jp/p/6-2
```

### ステップ 3: 実行
ターミナルで以下のコマンドを実行します。

```Bash
# 既存の古いコンテナを掃除してビルド・実行
docker-compose down
docker-compose up --build
```

### ステップ 4: 接続確認
実行後、ターミナルに以下の内容が表示されることを確認してください。
```
Initialization Sequence Completed: VPN接続が成功。
```
現在のIPアドレスを確認します: ここで表示されるIPが、自身の本来のIPではなくVPNサーバーのものになっていれば成功です。

## 4. トラブルシューティング
### DNS解決エラー (NameResolutionError)
VPN接続後にサイトの名前解決ができない場合、docker-compose.yml の dns セクションに 8.8.8.8 (Google) や 1.1.1.1 (Cloudflare) が設定されているか確認してください。

### 通信のフリーズ
接続は完了するが curl 等が返ってこない場合、パケットサイズがDockerネットワークの上限を超えている可能性があります。entrypoint.sh 内の --mssfix 1360 の数値を調整してください。

## 5. 注意・免責事項
### 5.1 法的および倫理的遵守
1. 対象サイトの利用規約: スクレイピングを実行する前に、対象サイトの robots.txt および利用規約を確認してください。短時間での大量アクセスは、サイトへの攻撃とみなされる可能性があります。

2. 著作権: 取得したデータの二次利用については、各国の著作権法を遵守してください。

### 5.2 セキュリティとプライバシー
1. IP漏洩の可能性: 本システムはVPN経由の通信を意図していますが、Dockerの挙動やIPv6の干渉により、100%の秘匿性を保証するものではありません。

1. 認証情報の管理: vpn/pass.txt にはプレーンテキストでパスワードが記載されます。このファイルをGitHub等の公開リポジトリにコミットしないよう、十分注意してください。

### 5.3 Proton VPNの利用規約
無料サーバーは混雑により予告なく切断されることがあります。また、一部のサイトではVPN経由のアクセスが既に遮断されている場合があります。

本システムを利用したことによって生じた、対象サイトへの損害、アカウントの停止、法的トラブル、およびその他のいかなる損失について、開発者は一切の責任を負いません。
